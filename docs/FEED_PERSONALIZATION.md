# Персонализация фида на основе истории просмотров

## Обзор

Система персонализации фида использует историю просмотров и взаимодействий пользователя для улучшения релевантности контента в ленте.

## Компоненты персонализации

### 1. Анализ времени просмотра

Система отслеживает время, которое пользователь проводит, просматривая каждый пост:

- **Для фото/текста/рецептов**: 3+ секунды = хороший просмотр
- **Для видео (reels)**: 50%+ просмотра = хороший просмотр
- **Пропуски**: < 0.5 секунды = быстрый скролл (negative feedback)

**Использование:**
- Хорошо просмотренные посты не показываются повторно (штраф 70%)
- Средне просмотренные посты показываются реже (штраф 30%)
- Пропущенные посты получают штраф 50%

### 2. Паттерны активности

Система анализирует паттерны активности пользователя:

- **Предпочтительные часы**: Топ-3 часа дня, когда пользователь наиболее активен
- **Среднее время просмотра**: Средняя длительность просмотра постов
- **Количество пропусков**: Сколько постов было пропущено

**Использование:**
- Буст 10% для постов, показанных в предпочтительные часы
- Буст 5% для постов, показанных в близкое время (±1 час)

### 3. Взаимодействие с авторами

Система отслеживает взаимодействие пользователя с конкретными авторами:

- **Лайки**: Вес 3.0
- **Сохранения**: Вес 2.5
- **Комментарии**: Вес 4.0
- **Хорошие просмотры** (3+ секунды): Вес 1.0

**Использование:**
- Буст до 20% для постов от авторов, с которыми пользователь активно взаимодействует
- Штраф 30% для авторов, посты которых часто пропускаются (>50% пропусков)
- Буст 10% для новых авторов (которых пользователь еще не видел)

### 4. Предпочтения по типам постов

Система анализирует, какие типы постов пользователь предпочитает:

- Анализ лайков и сохранений по типам (photo, recipe, reel, text)
- Взвешенная сумма: лайки (70%) + сохранения (30%)

**Использование:**
- Буст до 15% для предпочитаемых типов постов

### 5. Улучшенный расчет интересов

Интересы пользователя теперь учитывают веса:

- **Лайки**: Вес 3.0 за тег
- **Сохранения**: Вес 2.5 за тег
- **Хорошие просмотры**: Вес зависит от времени (до 2.0)

**Использование:**
- Более точное определение релевантности постов по тегам
- Учет интенсивности интереса (не просто наличие, а частота)

## Формула персонализации

```
personalization_boost = 
    author_interaction_boost *      # До 20% за активное взаимодействие
    post_type_preference_boost *    # До 15% за предпочитаемый тип
    skip_penalty *                  # 30% штраф за часто пропускаемых авторов
    new_author_boost                 # 10% за новых авторов
```

## Методы персонализации

### `_calculate_personalization_boost()`

Вычисляет общий boost персонализации на основе всех факторов.

### `_get_author_interaction_score()`

Вычисляет score взаимодействия с автором на основе:
- Лайков постов автора
- Сохранений постов автора
- Комментариев к постам автора
- Хороших просмотров постов автора

### `_is_author_frequently_skipped()`

Проверяет, часто ли пользователь пропускает посты автора (>50% пропусков).

### `_has_user_viewed_author_posts()`

Проверяет, просматривал ли пользователь посты автора ранее.

### `_get_user_viewing_patterns()`

Анализирует паттерны просмотров пользователя:
- Среднее время просмотра
- Предпочтительные часы активности
- Количество пропусков

### `_calculate_time_based_boost()`

Вычисляет boost на основе времени суток и активности пользователя.

### `_calculate_view_engagement()`

Вычисляет вовлеченность на основе времени просмотра поста.

### `_has_user_skipped_post()`

Проверяет, пропустил ли пользователь пост (быстрый скролл < 0.5 сек).

## Интеграция с ранжированием

Персонализация интегрирована в процесс ранжирования:

```python
score = base_score * personalization_boost
```

Где:
- `base_score` - базовый score из алгоритма ранжирования
- `personalization_boost` - boost персонализации (0.7 - 1.3)

## Примеры

### Высокий персонализационный boost
- Пост от автора, с которым пользователь активно взаимодействует
- Предпочитаемый тип поста (например, рецепты)
- Показан в предпочтительное время (например, вечером)
- Новый автор, которого пользователь еще не видел

### Низкий персонализационный boost
- Пост от автора, посты которого часто пропускаются
- Непредпочитаемый тип поста
- Показан в неактивное время
- Пост уже был хорошо просмотрен ранее
