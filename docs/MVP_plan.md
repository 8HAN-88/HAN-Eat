## H.A.N. Eat — MVP план (1 неделя)

### 1. Визия и цели
- **Цель продукта:** помочь пользователю готовить вкусные и полезные блюда из доступных ингредиентов, объединяя AI‑рекомендации, сообщество и минимум усилий.
- **Цели MVP:** доказать ценность персонализированных рекомендаций и UGC, обеспечить стабильный основной флоу (рекомендации → поиск → подробный рецепт → избранное/история) и подготовить инфраструктуру для монетизации.

### 2. Целевая аудитория и сценарии
- **Домашние кулинары / семьи (25‑45 лет):** быстрые полезные блюда, экономия бюджета.
- **Студенты / занятые профи (18‑30 лет):** “что приготовить из того, что есть в холодильнике”.
- **Фитнес‑энтузиасты (20‑40 лет):** контроль калорий/БЖУ, тренды ЗОЖ.

Ключевые сценарии (MVP):
1. Получить рекомендации по трендам → выбрать рецепт → посмотреть ингредиенты и шаги → поделиться или сохранить.
2. Ввести ингредиенты → получить выдачу → подобрать рецепт с минимумом доп. покупок.
3. Просмотреть историю запросов → повторить поиск.
4. Перейти в раздел сообщества → просмотреть карточки UGC → открыть видео.
5. Зарегистрироваться (email/Google) → сохранить избранное → загрузить собственный рецепт‑видео (в конец спринта).

### 3. Структура приложения (Flutter)
```
lib/
 ├─ app/                (инициализация, тема, маршруты, DI)
 ├─ core/               (константы, ошибки, utils, дизайн‑система)
 ├─ features/
 │   ├─ onboarding/
 │   ├─ auth/
 │   ├─ feed/           (AI рекомендации + поиск)
 │   ├─ recipe_detail/
 │   ├─ community/
 │   ├─ camera_scan/
 │   ├─ favorites/
 │   ├─ history/
 │   └─ settings/
 ├─ services/           (OpenAI, Spoonacular, Firebase, Stripe)
 └─ data/               (репозитории, модели, локальное хранилище)
```
- **Состояние:** Riverpod + Freezed/JsonSerializable для моделей.
- **Навигация:** go_router (поддержка deep link’ов).
- **UI:** Material 3 + кастомная минималистичная дизайн‑система (цветовые токены: светлый фон, акцент #FF7A00 для CTA, зеленый/желтый/красный индикаторы сложности).

### 4. Интеграции и сервисы
- **OpenAI:** парсинг ингредиентов, генерация рекомендаций, модерация UGC.
- **Spoonacular (или Edamam):** поиск рецептов, калории, рейтинги.
- **Firebase:** Auth (email + Google), Firestore (UGC метаданные, профили), Storage (видео), Analytics, Remote Config (AB). Cloud Functions для модерации/выплат.
- **SQLite (sqflite + drift/hive):** история, избранное, кэш карточек, настройки.
- **ML Kit / TensorFlow Lite:** анализ фото блюда из камеры (estimate calories).
- **Stripe/PayPal:** подписка + выплаты авторам (через serverless proxy).
- **Video:** `video_player` + `chewie` для UX, поддержка HLS/MP4.

### 5. Ключевые экраны и UX
- **Onboarding:** 3‑5 свайпов, CTA “Попробовать бесплатно”, “Войти / Регистрация”.
- **Домашний экран (Feed):** фиксированная строка поиска, AI‑рекомендации в бесконечной вертикальной сетке (2 столбца < 768px, 3 столбца >= 768px). Pull‑to‑refresh + skeleton‑placeholder’ы.
- **Карточка рецепта:** фото, название, время, сложность (цветная точка), калории, рейтинг/отзывы, CTA “Открыть”.
- **Поиск:** автодополнение ингредиентов, голосовой ввод (Speech‑to‑Text), фильтры (калории, сложность, диета, включить UGC).
- **Recipe detail:** hero‑анимация главного фото, параметры, ингредиенты (чек‑лист), шаги (нумерованные, с фото). Кнопки “Избранное”, “Поделиться”, “Редактировать (Premium)”, “Сканировать блюдо”.
- **Camera scan:** полноэкранный режим, результат “~450 ккал, Белки 20г, Жиры 15г, Углеводы 45г” + советы.
- **Сообщество:** отдельный таб, карточки UGC с превью видео, прогресс‑бар просмотра, лайки/комменты, badge “От сообщества”.
- **История / Меню / Настройки:** нижняя навигация (History, Menu, Community, Settings). Настройки включают аллергии, диеты, уведомления, профиль, подписку, выплаты.

### 6. Данные и модели
- **RecipeSummary:** id, title, heroImage, cookTime, difficulty, calories, rating, reviewsCount, isCommunity, source, tags.
- **RecipeDetail:** summary + ingredients[], steps[], media[], nutrition, affiliateLinks[].
- **UserProfile:** uid, name, avatar, allergies[], diets[], premium, payouts, stats.
- **SearchHistoryEntry:** query, timestamp, filters.
- **UGCVideo:** id, authorId, title, ingredients, steps, videoUrl, duration, status, metrics, payouts.

### 7. Офлайн и кэш
- Последние рекомендации и рецепт детально (json) + изображения (cached_network_image).
- История и избранное доступны offline.
- При отсутствии сети — баннер “Нет подключения”, предлагаем открыть кэшированные рецепты.

### 8. Монетизация
- **Freemium:** бесплатный доступ с баннерами и вставками спонсоров в карточках.
- **Premium ($3/мес, $30/год):** без рекламы, приоритетные рекомендации, оффлайн избранное, расширенная персонализация, доступ к инструментам анализа, возможность монетизировать UGC.
- **Affiliate:** кнопка “Добавить в шопинг‑лист” с deeplink в магазины.
- **UGC payouts:** фонд 0.02‑0.04$/1000 просмотров, Stripe Connect; комиссия приложения 20‑50%.

### 9. Безопасность и комплаенс
- Сбор согласий (GDPR/CPRA), отдельное хранение чувствительных данных (аллергии).
- Шифрование локальной БД (sqflite + SQLCipher) и защищённые ключи через secure storage.
- Возрастное ограничение 18+ для выплат, KYC для авторов (Stripe).
- Модерация контента: OpenAI + ручной review в Firebase admin. Логирование решений.

### 10. Тестирование и качество
- **Unit tests:** репозитории, use cases, провайдеры состояния.
- **Widget tests:** карточки, списки, пустые состояния.
- **Integration tests:** ключевые флоу (рекомендации → детали, поиск, UGC).
- **Device lab:** минимум Android (Pixel 5, Samsung A52), iOS (iPhone 13, SE 2). Проверка камеры, видео, оффлайн.
- **Analytics события:** install, onboardingComplete, searchQuery, recipeOpen, favoriteToggle, communityView, videoUpload, premiumPurchase, payoutRequest.
