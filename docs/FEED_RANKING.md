# Алгоритм ранжирования фида - Документация

## Обзор

Система ранжирования фида использует гибридный подход: rule-based алгоритм с множеством сигналов для определения релевантности постов для каждого пользователя.

## Формула ранжирования

```
score = (
    w1 * user_similarity +      # 30% - схожесть интересов
    w2 * recency +              # 20% - свежесть поста
    w3 * engagement +           # 30% - вовлеченность
    w4 * author_score +         # 15% - репутация автора
    w5 * additional_signals     # 5% - дополнительные сигналы
) * business_boost
```

## Компоненты

### 1. User Similarity (30%)

Вычисляет схожесть интересов пользователя с постом на основе:

- **Теги (40%)**: Общие теги между интересами пользователя и постом
  - Интересы извлекаются из лайков, сохранений и просмотров за последние 90 дней
  - Чем больше общих тегов, тем выше similarity

- **Подписка на автора (30%)**: Если пользователь подписан на автора
  - Boost: +1.0 к similarity

- **История взаимодействий (20%)**: Лайки постов этого автора
  - 5+ лайков постов автора = максимальный score

- **Тип поста (10%)**: Предпочтения пользователя по типам постов
  - Анализируются лайки и сохранения по типам (photo, recipe, reel, text)
  - Взвешенная сумма: лайки (70%) + сохранения (30%)

### 2. Recency (20%)

Экспоненциальное затухание по времени:

```
recency = exp(-hours_ago / 24)
```

- Half-life: 24 часа
- Новые посты получают максимальный score
- Старые посты постепенно теряют приоритет

### 3. Engagement (30%)

Вовлеченность поста на основе метрик:

- **Like Rate (40%)**: `likes / views`
- **Comment Rate (30%)**: `comments / views`
- **Save Rate (20%)**: `saves / views`
- **Repost Rate (10%)**: `reposts / views`

Используется логарифмическая нормализация для плавного роста:
```
like_score = min(log1p(like_rate * 100) / 5, 1.0)
```

### 4. Author Score (15%)

Репутация автора на основе:

- **Подписчики (30%)**: Логарифмическая нормализация
  - ~3000 подписчиков = 1.0

- **Средняя вовлеченность (50%)**: Средняя engagement постов автора за 30 дней
  - Анализируются последние 50 постов

- **Скорость роста (20%)**: Новые подписчики за последние 7 дней
  - 10+ новых подписчиков = 1.0

- **Верификация**: 10% boost для верифицированных авторов

### 5. Additional Signals (5%)

Дополнительные сигналы:

- **Новые посты**: 10% boost для постов, которые пользователь еще не видел
- **Посты от подписок**: 15% boost для постов от авторов, на которых подписан пользователь
- **Репосты от подписок**: 10% boost для репостов от подписок

### 6. Business Boost (множитель)

Бизнес-логика:

- **Plus авторы**: 30% boost для постов от Plus подписчиков
- **Каналы**: 20% boost для постов каналов, в которых пользователь состоит
- **Промо-посты**: (TODO) 50% boost для платных промо-постов

## Оптимизации

### Кэширование

- **User Interests**: Кэшируются на 1 час
- **Author Scores**: Кэшируются на 30 минут
- **Engagement метрики**: Обновляются в реальном времени из Redis

### Производительность

- Используются индексы БД для быстрых запросов
- Батчинг запросов для метрик
- Ленивая загрузка данных

## Будущие улучшения (ML)

### Планируемые улучшения

1. **Машинное обучение**:
   - Обучение модели на исторических данных
   - Использование embeddings для тегов и контента
   - Персонализация на основе поведения пользователя

2. **A/B тестирование**:
   - Тестирование разных весов компонентов
   - Оптимизация для разных сегментов пользователей

3. **Временные паттерны**:
   - Учет времени суток и дня недели
   - Адаптация к активности пользователя

4. **Социальные сигналы**:
   - Взаимные подписки
   - Общие интересы с друзьями
   - Популярность в социальном графе

## Метрики для оценки

- **CTR (Click-Through Rate)**: Процент просмотров постов
- **Engagement Rate**: Процент взаимодействий (лайки, комментарии)
- **Time Spent**: Время, проведенное в приложении
- **Retention**: Возвращаемость пользователей

## Настройка весов

Веса компонентов можно настроить через конфигурацию:

```python
FEED_RANKING_WEIGHTS = {
    "similarity": 0.3,
    "recency": 0.2,
    "engagement": 0.3,
    "author_score": 0.15,
    "additional_signals": 0.05
}
```

## Примеры

### Высокий score
- Пост от подписки
- С общими тегами с интересами пользователя
- Высокая engagement (много лайков/комментариев)
- От популярного автора
- Опубликован недавно

### Низкий score
- Пост от незнакомого автора
- Нет общих тегов
- Низкая engagement
- Опубликован давно
- От нового автора
