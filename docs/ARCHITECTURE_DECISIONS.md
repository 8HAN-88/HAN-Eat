# Архитектурные решения

## 1. Почему FastAPI/NestJS?

**FastAPI (Python):**
- Быстрая разработка
- Автоматическая документация (Swagger)
- Асинхронность из коробки
- Хорошая экосистема для ML (если понадобится для рекомендаций)

**NestJS (Node.js):**
- TypeScript из коробки
- Модульная архитектура
- Большая экосистема
- Хорошая производительность

**Решение:** Начать с FastAPI, так как уже есть Python код в проекте.

## 2. Почему PostgreSQL?

- ACID транзакции
- JSONB для гибких данных (body поста)
- Полнотекстовый поиск
- Массивные типы (tags)
- Проверенное решение для социальных сетей

## 3. Почему Redis?

- Быстрый кэш для лент
- Счетчики (likes, views)
- Rate limiting
- Очереди задач (Redis Streams)

## 4. Fanout Strategy

**Проблема:** Как распространять посты в ленты подписчиков?

**Решение:**
- **Fanout-on-write** для авторов с < 1000 подписчиков
  - При публикации сразу добавляем в ленты всех подписчиков
  - Просто, но не масштабируется для больших авторов
  
- **Fanout-on-read** для больших авторов
  - При чтении ленты собираем посты от подписок
  - Кэшируем последние посты автора
  - Масштабируется лучше

**Гибридный подход:**
- Используем fanout-on-write для малых авторов
- Для больших авторов используем fanout-on-read с кэшированием

## 5. Feed Ranking

**Phase 0 (Rule-based):**
- Простая формула с весами
- Быстро реализовать
- Легко отлаживать

**Phase 1 (ML):**
- Собираем данные о взаимодействиях
- Обучаем модель (LightGBM/XGBoost)
- A/B тестирование

**Phase 2 (Online Learning):**
- Модель обновляется в реальном времени
- Персонализация для каждого пользователя

## 6. Модерация

**Автоматическая:**
- OpenAI Moderation API для текста
- Custom ML для изображений (NSFW)
- Правила для рецептов (аллергены)

**Ручная:**
- Очередь для модераторов
- Приоритеты (жалобы > авто-флаги)
- История решений

## 7. Медиа Обработка

**Загрузка:**
- Presigned URLs для прямой загрузки в S3
- Клиент загружает напрямую, минуя backend
- Backend только координирует

**Обработка:**
- Очередь задач (RabbitMQ/Redis)
- FFmpeg workers для видео
- Автомасштабирование workers

**CDN:**
- Cloudflare/CloudFront
- Кэширование на edge
- Географическое распределение

## 8. Масштабирование

**Database:**
- Read replicas для чтения
- Connection pooling
- Индексы на всех важных полях
- Партиционирование по датам (если нужно)

**Caching:**
- Redis для лент (TTL: 5-15 минут)
- CDN для медиа
- Application cache для частых запросов

**Load Balancing:**
- Nginx/HAProxy перед API
- Health checks
- Graceful shutdown

## 9. Безопасность

**Аутентификация:**
- JWT токены (access + refresh)
- Короткий срок жизни access token (30 мин)
- Длинный срок жизни refresh token (7 дней)
- Revocation list для токенов

**Валидация:**
- Проверка размера файлов
- Проверка типов файлов
- Rate limiting
- SQL injection protection (ORM)

**Хранение паролей:**
- bcrypt или argon2
- Минимум 12 раундов
- Salt для каждого пароля

## 10. Мониторинг

**Метрики:**
- Prometheus для сбора
- Grafana для визуализации
- Алерты на критические метрики

**Логирование:**
- Structured logging (JSON)
- Централизованное хранение (ELK/CloudWatch)
- Уровни: DEBUG, INFO, WARNING, ERROR

**Трассировка:**
- OpenTelemetry для распределенной трассировки
- Отслеживание запросов через все сервисы

## 11. Тестирование

**Unit Tests:**
- Покрытие сервисов и моделей
- Моки для внешних зависимостей

**Integration Tests:**
- Тесты API endpoints
- Тесты с реальной БД (test database)

**E2E Tests:**
- Критические пути пользователя
- Автоматизация через Flutter Driver

## 12. CI/CD

**Pipeline:**
1. Lint и форматирование
2. Unit тесты
3. Integration тесты
4. Build
5. Deploy to staging
6. E2E тесты на staging
7. Deploy to production (manual approval)

**Environments:**
- Development (локально)
- Staging (тестирование)
- Production (продакшн)

## 13. GDPR Compliance

**Права пользователей:**
- Экспорт данных (GET /users/me/export)
- Удаление аккаунта (DELETE /users/me)
- Удаление всех связанных данных

**Хранение:**
- Минимизация данных
- Шифрование чувствительных данных
- Политика хранения (удаление через N дней)

## 14. Монетизация
