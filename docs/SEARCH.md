# Полнотекстовый поиск - Документация

## Обзор

Система полнотекстового поиска использует возможности PostgreSQL для быстрого и релевантного поиска по постам и рецептам.

## Технологии

- **PostgreSQL Full-Text Search**: Использует `tsvector` и `tsquery` для полнотекстового поиска
- **GIN индексы**: Для быстрого поиска по индексированным текстовым полям
- **Русская морфология**: Поддержка русского языка через конфигурацию `russian`

## Индексы

Созданы следующие GIN индексы для оптимизации поиска:

1. **idx_posts_title_description_fts**: Поиск в title и description
2. **idx_posts_tags_fts**: Поиск в тегах
3. **idx_posts_fulltext_fts**: Комбинированный индекс для всех текстовых полей
4. **idx_posts_status_visibility**: Составной индекс для фильтрации по статусу и видимости

## API Endpoints

### GET /api/v1/search/posts

Полнотекстовый поиск по постам.

**Параметры:**
- `q` (required): Поисковый запрос (минимум 1 символ)
- `post_type` (optional): Тип поста (photo, recipe, reel, text)
- `author_id` (optional): ID автора для фильтрации
- `channel_id` (optional): ID канала для фильтрации
- `tags` (optional): Теги через запятую (посты должны содержать хотя бы один тег)
- `date_from` (optional): Начальная дата публикации (формат: YYYY-MM-DD)
- `date_to` (optional): Конечная дата публикации (формат: YYYY-MM-DD)
- `min_likes` (optional): Минимальное количество лайков
- `min_comments` (optional): Минимальное количество комментариев
- `sort_by` (optional): Сортировка (relevance, date, popularity) - по умолчанию relevance
- `limit` (optional): Количество результатов (по умолчанию 20, максимум 100)
- `offset` (optional): Смещение для пагинации (по умолчанию 0)

**Примеры:**
```
# Базовый поиск
GET /api/v1/search/posts?q=борщ&post_type=recipe&limit=20

# Поиск с фильтрами
GET /api/v1/search/posts?q=курица&tags=ужин,быстро&min_likes=10&sort_by=popularity

# Поиск по дате
GET /api/v1/search/posts?q=рецепт&date_from=2024-01-01&date_to=2024-12-31

# Поиск по автору и каналу
GET /api/v1/search/posts?q=десерт&author_id=5&channel_id=3
```

**Ответ:**

}
```

### GET /api/v1/search/recipes

Специализированный поиск по рецептам.

**Параметры:**
- `q` (required): Поисковый запрос
- `author_id` (optional): ID автора для фильтрации
- `tags` (optional): Теги через запятую
- `date_from` (optional): Начальная дата публикации (формат: YYYY-MM-DD)

### GET /api/v1/search/suggestions

Получить предложения для автодополнения поиска.

**Параметры:**
- `q` (required): Поисковый запрос (минимум 1 символ)
- `limit` (optional): Количество предложений (по умолчанию 10, максимум 20)

**Пример:**
```

**Ответ:**
```json

    "борщ украинский",
    "борщ с мясом"
  ]

```

## Области поиска

### Для всех постов:
- **title**: Название поста
- **description**: Описание поста
- **tags**: Теги поста

### Для рецептов дополнительно:
- **body.ingredients**: Ингредиенты рецепта
- **body.steps**: Шаги приготовления

## Алгоритм поиска

1. **Подготовка запроса**: Преобразование пользовательского запроса в формат `tsquery`
   - Разбиение на слова
   - Экранирование специальных символов
   - Объединение через `&` (AND логика)

2. **Поиск**: Использование оператора `@@` для сопоставления `tsvector` с `tsquery`

3. **Ранжирование**: Использование `ts_rank_cd` для вычисления релевантности
   - Учитывает частоту вхождения слов
   - Учитывает близость слов
   - Сортировка по убыванию релевантности

4. **Фильтрация**: Применение фильтров по статусу, видимости, типу, автору

5. **Пагинация**: Возврат результатов с пагинацией

## Фильтры поиска

### Доступные фильтры

1. **По типу поста** (`post_type`)
   - `photo`: Фотографии
   - `recipe`: Рецепты
   - `reel`: Короткие видео
   - `text`: Текстовые посты

2. **По автору** (`author_id`)
   - Фильтрация по ID автора поста

3. **По каналу** (`channel_id`)
   - Фильтрация по ID канала/сообщества

4. **По тегам** (`tags`)
   - Список тегов через запятую
   - Пост должен содержать хотя бы один из указанных тегов

5. **По дате публикации** (`date_from`, `date_to`)
   - Формат: `YYYY-MM-DD`
   - Можно указать только начальную или только конечную дату

6. **По популярности** (`min_likes`, `min_comments`)
   - Минимальное количество лайков
   - Минимальное количество комментариев

### Сортировка

1. **relevance** (по умолчанию)
   - Сортировка по релевантности поискового запроса
   - Использует `ts_rank_cd` для ранжирования

2. **date**
   - Сортировка по дате публикации
   - Новые посты сначала

3. **popularity**
   - Сортировка по популярности
   - Сумма лайков и комментариев
   - Самые популярные посты сначала

## Особенности

### Русская морфология

PostgreSQL использует конфигурацию `russian` для:
- Стемминга (приведение к основе слова)
- Учета падежей
- Учета множественного числа

**Пример:**
Запрос "борщ" найдет:
- "борщ"
- "борща"
- "борщом"
- "борщи"

### Операторы поиска

- `&` (AND): Все слова должны присутствовать
- `|` (OR): Хотя бы одно слово должно присутствовать
- `!` (NOT): Исключить слово

**Примеры:**
- `борщ & мясо`: Найти посты, содержащие и "борщ", и "мясо"
- `борщ | суп`: Найти посты, содержащие "борщ" или "суп"
- `борщ ! вегетарианский`: Найти посты с "борщ", но без "вегетарианский"

### Релевантность

Релевантность вычисляется с помощью `ts_rank_cd`:
- Учитывает частоту вхождения слов
- Учитывает близость слов в тексте
- Учитывает позицию слов (слова в title важнее, чем в description)

## Производительность

### Оптимизации

1. **GIN индексы**: Быстрый поиск по индексированным полям
2. **Частичные индексы**: Индексы только для опубликованных постов
3. **Кэширование**: Кэширование популярных запросов (можно добавить Redis)

### Рекомендации

- Использовать пагинацию для больших результатов
- Ограничивать длину запроса (максимум 100 символов)
- Использовать фильтры для сужения результатов

## Ограничения

1. **Минимальная длина запроса**: 1 символ
2. **Максимальное количество результатов**: 100 за запрос
3. **Поддержка языков**: В основном русский язык (можно добавить другие)

## Будущие улучшения

1. **Мультиязычность**: Поддержка нескольких языков
2. **Фасетный поиск**: Фильтры по категориям, ингредиентам, времени приготовления
3. **Автодополнение**: Улучшенные предложения на основе популярных запросов
4. **Поиск по изображениям**: Поиск похожих изображений (ML)
5. **Персонализация**: Учет истории поиска пользователя
6. **Синонимы**: Поддержка синонимов (например, "томат" = "помидор")
