# ✅ Быстрый чек-лист готовности

## 🔴 Критически важно (без этого не запустится)

- [ ] Файл `.env` создан в корне проекта с `SPOONACULAR_API_KEY=ваш_ключ`
- [ ] `go_router` установлен (уже добавлен в `pubspec.yaml`, выполните `flutter pub get`)
- [ ] `google-services.json` добавлен в `android/app/` (для Android)
- [ ] `GoogleService-Info.plist` добавлен в `ios/Runner/` (для iOS)
- [ ] Плагин `com.google.gms.google-services` добавлен в `android/app/build.gradle.kts` (уже добавлен)

## 🟡 Важно (приложение запустится, но функции не будут работать)

- [ ] Бэкенд сервер запущен (`RecipeApp/app.py` или `app_simple.py`)
- [ ] Firebase проект создан и настроен
- [ ] API ключ Spoonacular получен и добавлен в `.env`

## 🟢 Желательно (для полной функциональности)

- [ ] Firebase Analytics настроен
- [ ] Firebase Cloud Messaging настроен для уведомлений
- [ ] Firestore правила безопасности настроены
- [ ] Firebase Storage правила настроены

## 🚀 Команды для запуска

```bash
# 1. Установить зависимости
flutter pub get

# 2. Сгенерировать код (если нужно)
flutter pub run build_runner build --delete-conflicting-outputs

# 3. Запустить бэкенд сервер (в отдельном терминале)
cd RecipeApp
python app.py

# 4. Запустить приложение
flutter run
```

## 📝 Проверка готовности

После выполнения всех пунктов из раздела "Критически важно", приложение должно:
- ✅ Запускаться без ошибок
- ✅ Показывать экран входа/регистрации
- ✅ Позволять искать рецепты (если бэкенд запущен)
- ✅ Сохранять избранное локально

## ⚠️ Если что-то не работает

1. Проверьте логи: `flutter run -v`
2. Проверьте что бэкенд сервер запущен и доступен
3. Проверьте что `.env` файл создан правильно
4. Проверьте что Firebase файлы на месте
5. Очистите и пересоберите: `flutter clean && flutter pub get`
