## ⚠️ Что может не работать без настройки:

- **Spoonacular API** - нужен `.env` файл с `SPOONACULAR_API_KEY`
  - Но бэкенд может работать и без него (с ограничениями)
  
- **Firebase функции** - нужна настройка Firebase
  - Авторизация, синхронизация избранного, уведомления
  - Но локальные функции работают без Firebase

## 🔧 Если бэкенд не запускается:

### Проблема: "ModuleNotFoundError"

**Решение:**
```bash
cd "D:\HAN Eat 1\RecipeApp"
pip install -r requirements.txt
```

Или используйте упрощенную версию:
```bash
python app_simple.py
```

### Проблема: "Port 5000 already in use"

**Решение:**
1. Найдите процесс на порту 5000 и закройте его
2. Или измените порт в `app.py` (строка с `app.run(port=5001)`)
3. И обновите URL в `lib/services/api_service.dart`

## 📝 Полная настройка (опционально)

### Для работы Spoonacular API:

1. Получите API ключ на https://spoonacular.com/food-api/console
2. Создайте файл `RecipeApp/.env`:
   ```
   SPOONACULAR_API_KEY=ваш_ключ_здесь
   ```

### Для работы Firebase:

См. инструкцию в `FIREBASE_SETUP.md`
