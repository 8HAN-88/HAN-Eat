# Исправление ошибки 404 для рекомендаций

## ✅ Что было исправлено:

1. **Создан новый endpoint `/recommendations`** в FastAPI бэкенде
   - Файл: `backend/app/api/v1/recipes.py`
   - Роутер подключен в `backend/app/main.py`

2. **Endpoint работает без API ключа Spoonacular**
   - Если ключ не указан, возвращается пустой список (без ошибки 404)
   - Если ключ указан, загружаются реальные рекомендации из Spoonacular API

## 🔧 Настройка (опционально):

Если вы хотите получать реальные рекомендации из Spoonacular API:

1. Получите API ключ на https://spoonacular.com/food-api/console
2. Добавьте в файл `backend/.env`:
   ```
   SPOONACULAR_API_KEY=ваш_api_ключ_здесь
   ```
3. Перезапустите бэкенд сервер

## 🚀 Проверка:

1. Убедитесь, что бэкенд сервер запущен
2. Откройте раздел Menu в приложении
3. Ошибка 404 должна исчезнуть
4. Если API ключ не указан, будет показан пустой список (без ошибки)
5. Если API ключ указан, будут показаны рекомендации рецептов

## 📝 Примечание:

Endpoint доступен по адресу: `http://localhost:5000/recommendations`

Можно проверить в браузере или через curl:
```bash
curl "http://localhost:5000/recommendations?limit=5"
```
