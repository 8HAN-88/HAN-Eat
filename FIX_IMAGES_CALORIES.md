# ✅ Исправление: Картинки и калории в Menu

## 🔧 Что было исправлено:

### 1. ✅ Добавлены калории в рекомендации

**Проблема:** В endpoint `/recommendations` не возвращались калории.

**Исправление:**
- Теперь для каждого рецепта получается детальная информация из Spoonacular API
- Извлекаются калории из поля `nutrition.nutrients`
- Калории добавляются в ответ API

**Файл:** `backend/app/api/v1/recipes.py` (строки 149-210)

### 2. ✅ Добавлены калории в поиск рецептов

**Проблема:** В endpoint `/recipes` (поиск) не возвращались калории.

**Исправление:**
- При получении детальной информации о рецепте извлекаются калории
- Калории добавляются в payload рецепта

**Файл:** `backend/app/api/v1/recipes.py` (строки 281-330)

### 3. ✅ Улучшено получение изображений

**Проблема:** Изображения могли отсутствовать или быть пустыми.

**Исправление:**
- Всегда получаем детальную информацию о рецепте
- Проверяем наличие изображения в детальной информации
- Используем `source_image` как fallback

## 📋 Формат данных:

Теперь каждый рецепт возвращает:
```json
{
  "id": 123,
  "title": "Recipe Title",
  "image": "https://...",
  "source_image": "https://...",
  "ingredients": [...],
  "steps": [...],
  "calories": 350,
  "nutrition": {...},
  ...
}
```

- **Производительность:** Детальная информация получается для каждого рецепта, что может замедлить загрузку (но это необходимо для калорий)
