# ✅ Исправлено: Картинки и калории в Menu

## 🔧 Что было исправлено:

### 1. ✅ Добавлены калории в рекомендации (`/recommendations`)

**Проблема:** Калории не возвращались из API.

**Исправление:**
- Для каждого рецепта получается детальная информация из Spoonacular API
- Извлекаются калории из поля `nutrition.nutrients` (ищем элемент с `name == "calories"`)
- Калории добавляются в ответ API

**Файл:** `backend/app/api/v1/recipes.py` (строки 149-210)

### 2. ✅ Добавлены калории в поиск рецептов (`/recipes`)

**Проблема:** Калории не возвращались при поиске.

**Исправление:**
- При получении детальной информации о рецепте извлекаются калории
- Калории добавляются в payload каждого рецепта

**Файл:** `backend/app/api/v1/recipes.py` (строки 281-330)

### 3. ✅ Улучшена обработка изображений

**Проблема:** Изображения могли быть пустыми строками вместо null.

**Исправление:**
- В бэкенде: пустые строки преобразуются в `None`
- Во фронтенде: улучшена обработка пустых строк в `Recipe.fromJson`
- Теперь пустые изображения корректно обрабатываются

**Файлы:**
- `backend/app/api/v1/recipes.py` (строки 196-210, 305-330)
- `lib/models/recipe.dart` (строки 54-60)

### 4. ✅ Улучшено получение детальной информации

**Изменения:**
- Для рекомендаций: всегда получаем детальную информацию о каждом рецепте
- Для поиска: получаем детальную информацию для калорий и полных данных
- Проверяем наличие изображения в детальной информации

## 📋 Формат данных:

Теперь каждый рецепт возвращает:
```json
{
  "id": 123,
  "title": "Recipe Title",
  "image": "https://spoonacular.com/recipeImages/123-312x231.jpg",
  "source_image": "https://spoonacular.com/recipeImages/123-312x231.jpg",
  "ingredients": ["ingredient1", "ingredient2", ...],
  "steps": [...],
  "calories": 350,
  "nutrition": {
    "nutrients": [...]
  },
  ...
}
```

## 🚀 Что делать:

1. **Перезапустите FastAPI бэкенд:**
   ```powershell
   cd "D:\HAN Eat 1\backend"
   python run.py
   ```

2. **Перезапустите Flutter приложение:**
   - Нажмите `R` для hot restart в терминале Flutter
   - Или перезапустите полностью: `flutter run`

3. **Проверьте Menu:**
   - Откройте раздел Menu
   - Должны отображаться картинки рецептов
   - Должны отображаться калории (если доступны в Spoonacular)

## ⚠️ Примечания:

- **Калории:** Могут быть не у всех рецептов (зависит от данных Spoonacular API)
- **Изображения:** Всегда должны быть у рецептов из Spoonacular
- **Производительность:** Детальная информация получается для каждого рецепта, что может немного замедлить загрузку (но это необходимо для калорий и полных данных)
- **Кэширование:** Результаты поиска кэшируются в Redis на 12 часов

## 🔍 Отладка:

Если картинки или калории все еще не отображаются:

1. **Проверьте логи бэкенда:**
   - Должны быть сообщения о получении детальной информации
   - Проверьте, что нет ошибок при запросах к Spoonacular

2. **Проверьте ответ API:**
   - Откройте `http://localhost:5000/docs`
   - Вызовите `GET /recommendations?limit=1`
   - Проверьте, что в ответе есть `image` и `calories`

3. **Проверьте консоль Flutter:**
   - Должны быть сообщения о загрузке изображений
   - Проверьте, нет ли ошибок при парсинге данных
