## 🔍 Диагностика:

### Шаг 1: Проверьте логи бэкенда

В терминале, где запущен `python run.py`, должны быть сообщения:
```
📋 Получено X рецептов из Spoonacular random API
🔍 Обработка рецепта 123: Recipe Title... image=True
✅ Recommendation 123: image=True, calories=350
     Image URL: https://spoonacular.com/recipeImages/...
📤 Возвращаем X рецептов с данными
```

**Если нет таких сообщений:**
- Проверьте, что бэкенд запущен
- Проверьте, что `SPOONACULAR_API_KEY` настроен

### Шаг 2: Проверьте API напрямую

**Откройте в браузере:**
```
http://localhost:5000/docs
```

**Вызовите:**
- `GET /recommendations?limit=1`

**Проверьте ответ:**
```json
{
  "recipes": [
    {
      "id": 123,
      "title": "...",
      "image": "https://...",
      "calories": 350,
      ...
    }
  ]
}
```

**Если `image` или `calories` отсутствуют:**
- Проверьте логи бэкенда на ошибки
- Проверьте, что `SPOONACULAR_API_KEY` правильный

### Шаг 3: Проверьте консоль Flutter

В терминале Flutter должны быть:
- Сообщения о загрузке изображений
- Или ошибки, если изображения не загружаются

**Если есть ошибки CORS:**
- Spoonacular должен разрешать загрузку изображений
- Если нет, можно добавить прокси на бэкенде

### Шаг 4: Проверьте Network в DevTools

1. Откройте DevTools (F12 в Chrome)
2. Перейдите на вкладку Network
3. Найдите запрос к `/recommendations`
4. Проверьте ответ - должны быть `image` и `calories`
5. Проверьте запросы к изображениям - должны загружаться

## 🔧 Возможные проблемы и решения:

### Проблема 1: SPOONACULAR_API_KEY не настроен

**Симптомы:**
- В логах: "Error fetching recipe details"
- API возвращает пустые рецепты

**Решение:**
1. Проверьте `backend/.env`:
   ```env
   SPOONACULAR_API_KEY=ваш_ключ_здесь
   ```
2. Перезапустите бэкенд

### Проблема 2: Spoonacular API не возвращает nutrition

**Симптомы:**
- В логах: `calories=None`
- В ответе API: `"calories": null`

**Решение:**
- Это нормально для некоторых рецептов
- Калории будут отображаться как "—" в UI

### Проблема 3: Изображения не загружаются

**Симптомы:**
- В логах: `image=True`, но в UI не отображается
- В консоли Flutter: ошибки загрузки изображений

**Решение:**
1. Проверьте URL изображения в ответе API
2. Проверьте, что URL доступен (откройте в браузере)
3. Проверьте CORS настройки

### Проблема 4: Данные не доходят до фронтенда

**Симптомы:**
- API возвращает данные, но UI не обновляется

**Решение:**
1. Проверьте Network в DevTools
2. Проверьте, что запрос идет на правильный URL
3. Перезапустите Flutter приложение
