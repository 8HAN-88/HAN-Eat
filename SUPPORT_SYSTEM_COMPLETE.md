## Процесс отмены подписки

1. **Пользователь запрашивает отмену** через `POST /subscriptions/cancel`
2. **Создается обращение** в поддержку типа `cancel_subscription`
3. **Админ обрабатывает обращение** через `POST /support/tickets/{id}/resolve`
4. **Автоматическая отмена** - при обработке подписка автоматически отменяется
5. **Подписка остается активной** до даты истечения (expires_at)

## Типы обращений

1. **cancel_subscription** - запрос на отмену подписки
2. **technical_issue** - техническая проблема
3. **billing** - вопросы по оплате
4. **other** - другие вопросы

## Особенности

1. **Защита от дубликатов** - нельзя создать два открытых обращения на отмену
2. **Автоматическая обработка** - отмена подписки происходит автоматически при обработке обращения
3. **Связь с сущностями** - обращения могут быть связаны с подписками, постами, пользователями
4. **История обращений** - пользователь видит все свои обращения
5. **Админ панель** - админы видят все обращения и могут их обрабатывать
