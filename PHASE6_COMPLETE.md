## Выполнено

### Backend

1. ✅ **Модель Repost**
   - Таблица `reposts` с уникальным ограничением (user_id, post_id)
   - Поле `comment` для комментария к репосту
   - Миграция `004_add_reposts.py`

2. ✅ **API для репостов**
   - `POST /api/v1/posts/{id}/repost` - создать репост (с опциональным комментарием)
   - `DELETE /api/v1/posts/{id}/repost` - удалить репост
   - `GET /api/v1/posts/{id}/reposts` - список пользователей, которые репостнули
   - `GET /api/v1/posts/{id}/is_reposted` - проверить, репостнул ли пользователь

3. ✅ **Интеграция с FeedService**
   - Репосты подписок включаются в ленту
   - Обогащение постов метаданными (reposts_count, is_reposted)

## Особенности

1. **Уникальность** - один пользователь может репостнуть пост только один раз
2. **Комментарии** - можно добавить комментарий к репосту
3. **Лента** - репосты подписок появляются в ленте
4. **Визуальная обратная связь** - иконка меняется при репосте
5. **Защита** - нельзя репостнуть свой пост
