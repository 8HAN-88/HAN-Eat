# Navigation & Analytics Integration - Завершено ✅

## Выполнено

### 1. ✅ Badge для уведомлений
- Добавлен badge с количеством непрочитанных уведомлений в навигации (RootShell)
- Badge отображается на иконке "Настройки" в нижней навигации
- Автоматическое обновление счетчика каждые 30 секунд
- Badge также отображается в Settings Screen на пункте "Уведомления"
- Обновление счетчика при переходе на экран уведомлений

### 2. ✅ Кнопка аналитики в Post Card
- Добавлена проверка, является ли текущий пользователь автором поста
- В меню поста (PopupMenuButton) добавлен пункт "Аналитика" для авторов
- Пункт "Пожаловаться" показывается только для не-авторов
- При клике на "Аналитика" открывается экран аналитики с параметром postId

### 3. ✅ Кнопка аналитики в Profile Screen
- Добавлена кнопка аналитики в AppBar для собственного профиля
- Добавлена кнопка "Аналитика" в header профиля рядом с "Редактировать"
- Кнопки отображаются только для собственного профиля (isOwnProfile)

## Структура изменений

### RootShell
- Преобразован из StatelessWidget в ConsumerStatefulWidget
- Добавлена загрузка и обновление счетчика непрочитанных уведомлений
- Badge отображается на иконке настроек

### NewPostCard
- Добавлена проверка текущего пользователя
- Добавлен пункт "Аналитика" в меню для авторов
- Условное отображение пунктов меню

### ProfileScreen
- Добавлена кнопка аналитики в AppBar
- Добавлена кнопка аналитики в header профиля

### SettingsScreen
- Преобразован из ConsumerWidget в ConsumerStatefulWidget
- Добавлена загрузка счетчика непрочитанных уведомлений
- Badge отображается на пункте "Уведомления"

## Особенности реализации

### Badge для уведомлений
- Используется виджет Badge из Material 3
- Отображается только если есть непрочитанные уведомления
- Формат: число или "99+" для больших значений
- Автоматическое обновление каждые 30 секунд

### Кнопки аналитики
- Проверка прав доступа (только для авторов)
- Навигация через go_router с параметрами
- Иконки: Icons.analytics_outlined
