# 🚀 Как запустить и проверить приложение HAN Eat

## 📋 Быстрая проверка готовности

### 1. Проверьте что зависимости установлены

```bash
flutter pub get
```

Если видите "Got dependencies!" - всё хорошо ✅

### 2. Проверьте доступные устройства

```bash
flutter devices
```

Вы увидите список доступных устройств (эмуляторы, физические устройства, браузер).

---

## 🎯 Варианты запуска

### Вариант 1: Запуск на Android (рекомендуется)

#### Если у вас есть Android эмулятор или устройство:

```bash
flutter run
```

Flutter автоматически выберет доступное Android устройство.

#### Если нужно выбрать конкретное устройство:

```bash
flutter devices  # Посмотрите список
flutter run -d <device-id>  # Замените <device-id> на ID устройства
```

### Вариант 2: Запуск в браузере (самый быстрый способ проверить)

```bash
flutter run -d chrome
```

⚠️ **Ограничения:** Некоторые функции (камера, уведомления) могут не работать в браузере.

### Вариант 3: Запуск на Windows

```bash
flutter run -d windows
```

---

## 🔍 Что проверить после запуска

### ✅ Основные функции:

1. **Экран меню открывается**
   - Должен показаться главный экран с поиском и рекомендациями

2. **Навигация работает**
   - Попробуйте переключиться между вкладками (История, Меню, Сообщество, Настройки)

3. **Поиск рецептов**
   - Введите ингредиенты в поиск
   - ⚠️ Если бэкенд не запущен, поиск будет использовать только локальный кэш

4. **Открытие деталей рецепта**
   - Нажмите на карточку рецепта
   - Должен открыться экран с деталями

5. **Избранное**
   - Нажмите на сердечко на карточке рецепта
   - Рецепт должен добавиться в избранное

### ⚠️ Что может не работать без настройки:

- **Firebase функции** (авторизация, синхронизация) - если не настроен Firebase
- **Поиск через Spoonacular** - если нет `.env` файла с API ключом
- **Бэкенд API** - если сервер не запущен

## 🐛 Решение проблем

### Ошибка: "No devices found"

**Решение:**
1. Для Android: Запустите Android Studio → AVD Manager → Create/Start эмулятор
2. Для браузера: Используйте `flutter run -d chrome`
3. Для Windows: Используйте `flutter run -d windows`

### Ошибка: "Build failed"

**Решение:**
```bash
flutter clean
flutter pub get
flutter run
```

### Ошибка: "go_router not found"

**Решение:**
```bash
flutter pub get
```

### Ошибка: "Firebase not initialized"

**Решение:**
- Это нормально, если Firebase не настроен
- Приложение будет работать, но без авторизации и синхронизации
- Для полной функциональности настройте Firebase (см. FIREBASE_SETUP.md)

## 📱 Полная проверка функциональности

### Шаг 1: Базовый запуск (без настройки)

```bash
flutter run -d chrome  # или flutter run для Android
```

**Проверьте:**
- ✅ Приложение запускается
- ✅ Навигация работает
- ✅ Экран меню отображается
- ✅ Можно открыть детали рецепта

### Шаг 2: С бэкенд сервером (для поиска рецептов)

1. **Запустите бэкенд сервер** (в отдельном терминале):
   ```bash
   cd RecipeApp
   python app.py
   ```

2. **Запустите приложение**:
   ```bash
   flutter run
   ```

3. **Проверьте:**
   - ✅ Поиск рецептов работает
   - ✅ Рекомендации загружаются

### Шаг 3: С Firebase (для полной функциональности)

1. **Настройте Firebase** (см. FIREBASE_SETUP.md)
2. **Запустите приложение**
3. **Проверьте:**
   - ✅ Авторизация работает
   - ✅ Избранное синхронизируется
   - ✅ Уведомления работают
