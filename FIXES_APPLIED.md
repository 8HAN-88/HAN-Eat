# ✅ Исправления применены

## 🔧 Исправленные проблемы:

### 1. TypeError в DetailPage (type 'int' is not a subtype of type 'String')

**Проблема:** В `detail_page.dart` при обработке шагов рецепта поле `step['step']` могло быть int вместо String.

**Исправление:**
- Добавлена проверка типа и преобразование в строку
- Теперь код корректно обрабатывает как `step['step']`, так и `step['instruction']`

**Файл:** `lib/screens/detail_page.dart` (строка 428-429)

### 2. 404 ошибка для комментариев к рецептам

**Проблема:** Endpoint `/recipes/{id}/comments` не существовал в FastAPI бэкенде.

**Исправление:**
- Добавлен `GET /recipes/{recipe_id}/comments` - получение комментариев
- Добавлен `POST /recipes/{recipe_id}/comments` - добавление комментария
- Пока возвращают пустые данные (функциональность не реализована полностью)

**Файл:** `backend/app/api/v1/recipes.py` (строки 487-500)

### 3. Исправлена структура шагов в parse_steps

**Проблема:** Функция `parse_steps` возвращала неправильную структуру данных.

**Исправление:**
- Теперь возвращает правильные поля: `number`, `step`, `instruction`
- Соответствует ожидаемой структуре во фронтенде

**Файл:** `backend/app/api/v1/recipes.py` (строки 27-40)
