# ⚠️ ВАЖНО: Проверка SPOONACULAR_API_KEY

## 🔍 Проблема:

В логах видно: **"📋 Получено 0 рецептов из Spoonacular"**

Это означает, что либо:
1. `SPOONACULAR_API_KEY` не настроен
2. API ключ неправильный или неактивен
3. Spoonacular API не возвращает рецепты

## ✅ Решение:

### 1. Проверьте, что ключ добавлен в `backend/.env`:

```env
SPOONACULAR_API_KEY=ваш_api_ключ_здесь
```

### 2. Перезапустите бэкенд:

```powershell
cd "D:\HAN Eat 1\backend"
python run.py
```

**В логах должно быть:**
```
✅ SPOONACULAR_API_KEY настроен (длина: XX)
```

**Если видите:**
```
⚠️ SPOONACULAR_API_KEY не настроен! Добавьте его в backend/.env
```

То ключ не настроен или не загружается.

### 3. Проверьте логи при запросе:

При запросе к `/recommendations` должны быть сообщения:
```
🌐 Запрос к Spoonacular: https://api.spoonacular.com/recipes/random с параметрами {...}
📡 Ответ Spoonacular: status=200
📋 Получено X рецептов из Spoonacular random API
```

**Если status != 200:**
- Проверьте, что API ключ правильный
- Проверьте, что ключ активен на Spoonacular
- Проверьте лимиты API (бесплатный план имеет ограничения)

### 4. Получите новый API ключ (если нужно):

1. Зайдите на https://spoonacular.com/food-api/console
2. Зарегистрируйтесь или войдите
3. Получите API ключ
4. Добавьте в `backend/.env`:
   ```env
   SPOONACULAR_API_KEY=ваш_новый_ключ
   ```
5. Перезапустите бэкенд

## 🚀 После настройки:

1. Перезапустите бэкенд
2. Проверьте логи - должны быть сообщения о получении рецептов
3. Проверьте API в `/docs` - должны возвращаться рецепты с `image` и `calories`
4. Перезапустите Flutter приложение

**Если проблема сохраняется, пришлите:**

- Сообщение о SPOONACULAR_API_KEY (настроен или нет)
