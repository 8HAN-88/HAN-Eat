# Исправление ошибки "Backend сервер не запущен" при регистрации/входе

## 🔍 Проблема:

При попытке регистрации или входа появляется ошибка: **"Backend сервер не запущен"**

## ✅ Решение:

### 1. Убедитесь, что FastAPI бэкенд запущен

**Важно:** Для регистрации и входа нужен **FastAPI бэкенд**, а не старый Flask сервер!

#### Запустите FastAPI бэкенд:

```powershell
cd "D:\HAN Eat 1\backend"
python run.py
```

Бэкенд должен запуститься на `http://localhost:5000`

### 2. Проверьте, что бэкенд работает

Откройте в браузере:
- `http://localhost:5000/health` - должен вернуть `{"status": "ok"}`
- `http://localhost:5000/docs` - должна открыться документация API

### 3. Остановите старый Flask сервер (если запущен)

Если у вас запущен старый Flask сервер (`RecipeApp/app.py`), остановите его:
- Нажмите `Ctrl+C` в терминале, где он запущен
- Или закройте этот терминал

**Важно:** На порту 5000 должен работать только **FastAPI бэкенд**!

## 📋 Endpoints для аутентификации:

FastAPI бэкенд предоставляет следующие endpoints:
- `POST /api/v1/auth/register` - регистрация
- `POST /api/v1/auth/login` - вход
- `POST /api/v1/auth/google` - вход через Google
- `POST /api/v1/auth/refresh` - обновление токена

## 🔧 Если проблема сохраняется:

1. **Проверьте, что бэкенд запущен:**
   ```powershell
   # Проверка порта 5000
   netstat -ano | findstr :5000
   ```

2. **Проверьте логи бэкенда:**
   - В терминале, где запущен `python run.py`, должны быть логи запросов

3. **Проверьте базу данных:**
   - Убедитесь, что PostgreSQL и Redis запущены (через Docker)
   - Проверьте, что миграции применены

4. **Проверьте URL в коде:**
   - `lib/services/auth_service.dart` использует `http://localhost:5000/api/v1`
   - Это правильно для FastAPI бэкенда

## ✅ После исправления:

1. Перезапустите Flutter приложение
2. Попробуйте зарегистрироваться или войти
3. Ошибка должна исчезнуть

---

**Примечание:** Для рекомендаций в Menu используется endpoint `/recommendations`, который уже работает. Для регистрации/входа нужен FastAPI бэкенд с endpoints `/api/v1/auth/*`.
