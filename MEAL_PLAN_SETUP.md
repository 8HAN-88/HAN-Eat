# 🍽️ Инструкция по настройке Плана питания

## ✅ Что реализовано

1. **Модели данных** (`lib/models/meal_plan.dart`):
   - `MealType` - типы приемов пищи (завтрак, обед, ужин, перекус)
   - `MealPlanEntry` - запись в плане питания
   - `DailyMealPlan` - план на день

2. **Сервис** (`lib/services/meal_plan_service.dart`):
   - Локальное хранение через Hive
   - Синхронизация с Firebase Firestore
   - Автоматические напоминания за 30 минут до приема пищи

3. **UI экраны**:
   - `MealPlanScreen` - просмотр плана на неделю
   - `AddToMealPlanScreen` - добавление рецептов в план

4. **Интеграция**:
   - Кнопка "Добавить в план" в `DetailPage`
   - Ссылка в настройках
   - Маршрут `/meal-plan`

## 🔧 Настройка

### 1. Установи зависимости

```bash
flutter pub get
```

### 2. Сгенерируй Hive адаптеры

```bash
flutter pub run build_runner build --delete-conflicting-outputs
```

Это создаст файл `lib/models/meal_plan.g.dart` с адаптерами для Hive.

### 3. Проверь инициализацию

Убедись что в `lib/app/bootstrap.dart` добавлено:
- Импорты моделей и сервиса
- Регистрация Hive адаптеров
- Инициализация `MealPlanService.init()`

## 📱 Использование

1. **Добавить рецепт в план**:
   - Открой детали рецепта
   - Нажми кнопку календаря в AppBar
   - Выбери дату, тип приема пищи и количество порций

2. **Просмотреть план**:
   - Зайди в Настройки
   - Нажми "План питания"
   - Выбери день недели
   - Просмотри запланированные блюда

3. **Удалить из плана**:
   - Открой план питания
   - Нажми на блюдо
   - Нажми кнопку удаления

## 🔔 Напоминания

Напоминания о приемах пищи приходят автоматически за 30 минут до запланированного времени.

## ⚠️ Важно

- После добавления новых моделей нужно запустить `build_runner` для генерации адаптеров
- План питания синхронизируется с Firebase при авторизации
- Локальные данные хранятся в Hive box `meal_plans`
