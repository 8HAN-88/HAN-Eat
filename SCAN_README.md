# Сканирование блюд — применение изменений

Чтобы изменения в скане (перевод, калории, БЖУ, картинки) заработали:

## 1. Перезапустите бэкенд

Код анализа фото находится на **бэкенде** (Python). После любых правок в папке `backend/` нужно перезапустить сервер.

```bash
# Остановите текущий процесс (Ctrl+C), затем снова:
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 5000
```

Или как вы обычно запускаете API на порту 5000.

## 2. Пересоберите приложение

После правок в коде Flutter (папка `lib/`) нужна новая сборка:

```bash
cd "d:\HAN Eat 1"
flutter clean
flutter pub get
flutter run -d chrome
# или flutter run -d windows
```

## 3. Язык в настройках

- Откройте **Настройки** в приложении.
- В блоке **«Язык перевода»** выберите нужный язык (Русский, English и т.д.).
- Сохраните (выбор сохраняется автоматически).
- После этого делайте сканирование — название блюда, рецепты и питательность будут на выбранном языке.

## Что было исправлено

- **Бэкенд (анализ через Spoonacular):** перевод названия блюда и рецептов на выбранный язык, приведение калорий и БЖУ к единому формату, поддержка разных форматов ответа API.
- **Приложение (анализ через GPT):** жёсткое требование выдавать название блюда на выбранном языке, корректный разбор чисел в питательности.
- **Настройки:** выпадающий список языка теперь показывает текущее значение (`value` вместо `initialValue`).

Если после перезапуска бэкенда и пересборки приложения сканирование всё ещё ведёт себя по-старому, проверьте, что приложение обращается к нужному серверу (в коде — `localhost:5000` для десктопа и веба).
